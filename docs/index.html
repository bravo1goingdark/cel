cel is a desktop app for creating ascii animations — the kind that appear in CLI spinners, terminal UIs, documentation banners, and retro web interfaces.

work with characters on a grid, not pixels. keyframe timeline, real-time preview, one-click export to seven formats.

[01] engine

zero runtime dependencies. works in node, deno, bun, any browser. the entire engine is 3.9KB gzipped.

three functions cover 90% of use cases. sampling uses binary search. serialization is deterministic — identical input, byte-identical output.

import { sampleScene, serialize, validate } from '@cel/core'; // sample all sprites at 1500ms const frame = sampleScene(scene, 1500); // deterministic json (git-friendly) const json = serialize(scene); // full validation with error paths const result = validate(input);
210 tests · 100% core coverage · 0.6ms per frame · 3.9KB gzipped

[04] desktop app

  • visual timeline — drag keyframes, scrub playhead
  • real-time preview — 60fps playback at any speed
  • sprite management — add, rename, duplicate, reorder
  • undo/redo — 30-entry stack with coalescing
  • autosave — 400ms debounce to tauri store
  • native menus — file, edit, view, help
  • atomic saves — write-tmp, fsync, rename-over
  • native dialogs — open, save, export
  • dark/light theme — system detection
  • 31 bundled presets — ready to use or remix

built with tauri + rust. sub-second launch. under 10MB installed.
